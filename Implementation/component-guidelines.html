Primary Button - Use for the main actions

The Primary button represents the most important action on a page or within a component. Use only one Primary button per view to maintain a clear visual hierarchy, excluding buttons on the page header.

Secondary Button - Use for supporting actions

The Secondary button is used for actions that are important but not the main call to action. It provides a clear alternative without competing visually with the Primary button.

Tertiary Button - Use for launching actions

The Tertiary button is to be used sparingly, and primarily for actions that connect or launch other features. Can also be used with a red variation in limited scenarios for canceling or deleting actions.

Default

The standard appearance of the button. Use this state for all active, ready‑to‑use actions.

Hover

Utilizes a dark background and colored border to indicate interactivity.

Pressed

Represents that the button has been pressed down.

Disabled

Represents that the button is not available for interaction.
